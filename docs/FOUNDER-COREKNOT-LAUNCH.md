# CoreKnot production launch — founder checklist

> **Aligned with:** [architecture/DEPLOYMENT-ARCHITECTURE.md](./architecture/DEPLOYMENT-ARCHITECTURE.md) · [architecture/MONGO-SUNSET-REPORT.md](./architecture/MONGO-SUNSET-REPORT.md)  
> **Target end state:** Vercel + Railway + Neon + Redis — **no Mongo, no Supabase.** Parallel-run may temporarily require `MONGODB_URI_PROD` (30 days).

Run this **after** migration code is on `main` and CI is green. Agents cannot set Dashboard secrets or DNS — this is your copy-paste runbook.

Rollback: [apps/coreknot/docs/DEPLOY_ROLLBACK.md](../apps/coreknot/docs/DEPLOY_ROLLBACK.md)

---

## 1. Neon Postgres

1. Open [Neon console](https://console.neon.tech) → production project → **main** branch.
2. Enable **PITR** (point-in-time recovery) on the production branch.
3. Copy **`DATABASE_URL`** (pooled connection string for Railway).
4. Locally (one time, against prod read-only Mongo if delta needed):

```powershell
pnpm db:generate
pnpm db:push
pnpm migrate:coreknot:p0:execute
pnpm migrate:coreknot:p1:execute
pnpm migrate:coreknot:count-parity -- --p1
```

Expect **WITHIN_TOLERANCE** before cutover.

---

## 2. Railway — two services

Monorepo root: repository root. Service root: `apps/coreknot/server`.

| Service | Start command | `RUN_WORKERS` |
|---------|---------------|---------------|
| **coreknot-api** | `node server.js` | `false` |
| **coreknot-worker** | `node workers/startWorkers.js` | `true` |

Use [`apps/coreknot/server/railway.toml`](../apps/coreknot/server/railway.toml) and env template [`railway.env.example`](../apps/coreknot/server/railway.env.example).

### Required secrets (both services)

| Variable | How to get |
|----------|------------|
| `DATABASE_URL` | Neon (step 1) |
| `REDIS_URL` | Upstash (step 3) |
| `JWT_SECRET` | `openssl rand -base64 48` |
| `ENCRYPTION_KEY` | `openssl rand -hex 32` |
| `RESEND_API_KEY` | Resend dashboard (step 4) |

### Non-secret env (paste on both)

COREKNOT_POSTGRES_ENABLED=true

Health check path: `/api/health/ready`

## 3. Upstash Redis

1. Create Redis database (same region as Railway if possible).
2. Copy **`REDIS_URL`** → both Railway services.

---

## 4. Resend

1. Verify sending domain (e.g. `coreknot.in` or `theshakticollective.in`).
2. Add **`RESEND_API_KEY`** to Railway API + worker.

---

## 5. Google OAuth (optional but recommended)

1. Google Cloud Console → OAuth client.
2. Redirect URI: `https://api.coreknot.in/api/auth/google/callback`
3. Set `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` on Railway API.

---

## 6. Vercel — client

1. Project root: **`apps/coreknot/client`**
2. Production env:

```env
RENDER_API_PROXY_URL=https://api.coreknot.in
```

3. Domains: `coreknot.in`, `www.coreknot.in`

Proxy config is generated by [`apps/coreknot/client/scripts/generateVercelConfig.cjs`](../apps/coreknot/client/scripts/generateVercelConfig.cjs).

## 9. Deploy & verify

1. Push `main` (or trigger Railway redeploy).
2. Redeploy **both** Railway services.
3. Redeploy Vercel production.

```powershell
curl https://api.coreknot.in/api/health/ready
pnpm migrate:coreknot:verify-cutover:ping
pnpm sweep:prod
```

Ready response should include `dependencies.postgres.ok: true`, `migration.fullPostgresCutover: true`.

### Manual smoke

- Login (email + Google if configured)
- CRM leads list
- Task board load
- Send test campaign → expect **202** from mail API

---

## 10. Rollback (if needed)

1. Re-enable Mongo: `COREKNOT_MONGO_REQUIRED=true`, set `MONGODB_URI_PROD`, flip affected `COREKNOT_*_STORE=mongo`.
2. Redeploy API + worker.
3. Follow [DEPLOY_ROLLBACK.md](../apps/coreknot/docs/DEPLOY_ROLLBACK.md).
